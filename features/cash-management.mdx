## Overview

The Cash & Accounting module (Caja & Contabilidad) provides comprehensive tracking of your business's financial transactions, including income, expenses, and cash flow management. This feature helps you maintain accurate financial records and monitor your business's financial health in real-time.

<Note>
  This module is different from the [Cash Register](/features/cash-register), which focuses on daily register operations. Cash Management provides broader accounting and financial tracking capabilities.
</Note>

## Financial Dashboard

The Cash Management module displays four key financial metrics:

### Income (Ingresos)
Total revenue from all sales and income transactions for the selected period.

```typescript
totalIncome = transactions
  .filter((t) => t.type === 'income')
  .reduce((sum, t) => sum + t.amount, 0);
```

### Expenses (Egresos)
Total outgoing payments including purchases, services, and other expenses.

```typescript
totalExpenses = transactions
  .filter((t) => t.type === 'expense')
  .reduce((sum, t) => sum + t.amount, 0);
```

### Net Balance
The difference between income and expenses, showing overall profitability.

```typescript
netFlow = totalIncome - totalExpenses;
```

### Payment Method Balances

**Cash Balance:**
```typescript
cashBalance = transactions
  .filter((t) => t.paymentMethod === 'efectivo')
  .reduce((sum, t) => sum + (t.type === 'income' ? t.amount : -t.amount), 0);
```

**Card Balance:**
```typescript
cardBalance = transactions
  .filter((t) => t.paymentMethod === 'tarjeta')
  .reduce((sum, t) => sum + (t.type === 'income' ? t.amount : -t.amount), 0);
```

## Transaction Types

### Income Transactions
- **Type**: `'income'`
- **Examples**: Sales, payments received, service revenue
- **Display**: Green/success badge
- **Impact**: Increases cash balance

### Expense Transactions
- **Type**: `'expense'`
- **Examples**: Inventory purchases, service payments, utilities
- **Display**: Red/danger badge
- **Impact**: Decreases cash balance

## Transaction Structure

Each transaction record includes:

```typescript
interface Transaction {
  id: string;
  type: 'income' | 'expense';
  description: string;
  amount: number;
  paymentMethod: 'efectivo' | 'tarjeta' | 'transferencia';
  date: string;
  category: string;
}
```

<ParamField path="id" type="string" required>
  Unique transaction identifier
</ParamField>

<ParamField path="type" type="'income' | 'expense'" required>
  Transaction type - income or expense
</ParamField>

<ParamField path="description" type="string" required>
  Description of the transaction (e.g., "Venta #001", "Compra inventario")
</ParamField>

<ParamField path="amount" type="number" required>
  Transaction amount in local currency
</ParamField>

<ParamField path="paymentMethod" type="string" required>
  Payment method used: efectivo (cash), tarjeta (card), or transferencia (transfer)
</ParamField>

<ParamField path="date" type="string" required>
  Transaction date and time (format: "YYYY-MM-DD HH:mm")
</ParamField>

<ParamField path="category" type="string" required>
  Transaction category (e.g., "ventas", "inventario", "servicios")
</ParamField>

### Negative Trends

## Integration with Other Modules

### Sales Integration
Income transactions are automatically created from:
- POS sales (from `/pages/pos`)
- Invoice payments (from `/pages/invoices`)

### Purchase Integration
Expense transactions are created from:
- Purchase orders (from `/pages/purchases`)
- Supplier payments (from `/pages/suppliers`)

### Cash Register Relationship
While the Cash Register tracks daily register operations, Cash Management provides:
- Longer-term financial tracking
- Expense categorization
- Multi-payment method analysis
- Trend analysis and reporting
