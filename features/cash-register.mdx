## Overview

The Cash Register module in Karma POS allows you to manage daily cash register operations, track opening and closing balances, and reconcile cash and card payments at the end of shifts.

## Cash Register Structure

### Cash Register Model

```typescript
interface CashRegister {
    id: string;
    name: string;
    status: "open" | "closed";
    openedAt: string;
    closedAt?: string;
    openingBalance: number;
    expectedBalance: number;
    actualBalance?: number;
    difference?: number;
    operator: string;
}
```

## Opening a Cash Register

### Open Register Dialog

```typescript
handleOpenCashRegister(): void {
    if (this.openingBalance === null || this.openingBalance <= 0) {
        // Show validation error
        return;
    }

    const newCashRegister: CashRegister = {
        id: String(Date.now()),
        name: `Caja ${this.cashRegisters.length + 1}`,
        status: "open",
        openedAt: new Date().toISOString(),
        openingBalance: this.openingBalance,
        expectedBalance: this.openingBalance,
        operator: "Usuario Actual",
    };

    this.cashRegisters = [newCashRegister, ...this.cashRegisters];
    console.log("Abriendo caja con balance inicial:", this.openingBalance);
    
    this.showOpenDialog = false;
    this.openingBalance = null;
    this.updateActiveCashRegister();
}
```

  <Step title="Assign Operator">

  <Step title="Register Opens">

  </Step>
</Steps>

<Note>

</Note>

## Managing Active Register

### Find Active Register

```typescript
updateActiveCashRegister(): void {
    this.activeCashRegister = this.cashRegisters.find(
        (cr) => cr.status === "open"
    );
}
```

Only one cash register can be active (open) at a time per user.

### Expected Balance Calculation

## Closing a Cash Register

### Close Register Process

```typescript
handleCloseCashRegister(): void {
    if (!this.activeCashRegister || 
        this.actualCash === null || 
        this.actualCard === null) {
        // Show validation error
        return;
    }

    const totalActual = this.actualCash + this.actualCard;
    const difference = totalActual - 
        (this.activeCashRegister.expectedBalance || 0);

    this.cashRegisters = this.cashRegisters.map((cr) =>
        cr.id === this.activeCashRegister?.id
            ? {
                ...cr,
                status: "closed",
                closedAt: new Date().toISOString(),
                actualBalance: totalActual,
                difference: difference,
            }
            : cr
    );
    
    console.log("Cerrando caja - Diferencia:", difference);
    
    this.showCloseDialog = false;
    this.actualCash = null;
    this.actualCard = null;
    this.updateActiveCashRegister();
}
```

  <Step title="Count Cash">
    Enter the actual cash amount in the register

  <Step title="Count Card Payments">

## Reconciliation

### Calculating Differences

The difference indicates discrepancies:

```typescript
const totalActual = this.actualCash + this.actualCard;
const difference = totalActual - this.activeCashRegister.expectedBalance;
```

**Interpretation:**
- **Positive difference**: More money than expected (overage)
- **Negative difference**: Less money than expected (shortage)
- **Zero difference**: Perfect balance

<Warning>
  Significant differences may indicate errors in transactions, theft, or miscounting. Investigate large variances immediately.
</Warning>

## Display & Formatting

### Date Formatting

```typescript
formatDate(dateString: string | undefined): string {
    if (!dateString) return '';
    return new Date(dateString).toLocaleString('es-CO', {
        year: 'numeric',
        month: 'short',
        day: 'numeric',
        hour: '2-digit',
        minute: '2-digit',
        second: '2-digit'
    });
}
```

### Time Open Calculation

```typescript
getOpenTime(openedAt: string): string {
    const openedDate = new Date(openedAt).getTime();
    const now = Date.now();
    const diffHours = Math.floor((now - openedDate) / (1000 * 60 * 60));
    return `${diffHours}h`;
}
```

## Audit Trail

Each cash register session creates an audit record:

- **Who**: Operator name
- **When**: Open and close timestamps
- **What**: All balances and differences
- **Status**: Open or closed

<Info>
  This audit trail is essential for accounting, theft prevention, and financial reporting.
</Info>
