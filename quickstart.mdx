# Quickstart Guide

Get Karma POS running on your local machine in just a few minutes. This guide will walk you through the essential steps to start the development server and access the application.

<Note>
  This quickstart assumes you have Node.js 18+ and npm installed. If not, download them from [nodejs.org](https://nodejs.org/) first.
</Note>

## Quick Setup in 4 Steps

<Steps>
  <Step title="Clone the Repository">
    Clone the Karma POS repository to your local machine:

    ```bash
    git clone https://github.com/your-organization/karma-pos.git
    cd karma-pos
    ```

    <Info>
      Replace `your-organization/karma-pos` with the actual repository URL provided by your organization.
    </Info>
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>

    This will install all dependencies including:
    - Angular 19 framework
    - PrimeNG UI components
    - SignalR for real-time communication
    - PDF generation libraries
    - Excel import/export tools

    <Tip>
      Installation typically takes 2-5 minutes depending on your internet connection.
    </Tip>
  </Step>

  <Step title="Start Development Server">
    Launch the Angular development server:

    ```bash
    ng serve
    ```

    Or use the npm script:

    ```bash
    npm start
    ```

    Wait for the compilation to complete. You'll see output like:

    ```bash
    ** Angular Live Development Server is listening on localhost:4200 **
    ✔ Compiled successfully.
    ```

    <Warning>
      The first compilation may take 30-60 seconds. Subsequent builds will be much faster with Angular's incremental compilation.
    </Warning>
  </Step>

  <Step title="Access the Application">
    Open your web browser and navigate to:

    ```
    http://localhost:4200
    ```

    You should see the Karma POS login screen. The application will automatically reload when you make changes to the source code.

    <Info>
      **Default Development Port**: The application runs on port 4200 by default. If this port is busy, Angular CLI will automatically use the next available port (4201, 4202, etc.).
    </Info>
  </Step>
</Steps>

## Troubleshooting Quick Fixes

<AccordionGroup>
  <Accordion title="Port 4200 Already in Use">
    If you see an error that port 4200 is already in use:

    ```bash
    # Option 1: Use a different port
    ng serve --port 4300
    
    # Option 2: Kill the process using port 4200
    # On Windows:
    netstat -ano | findstr :4200
    taskkill /PID <PID> /F
    
    # On macOS/Linux:
    lsof -ti:4200 | xargs kill -9
    ```
  </Accordion>

  <Accordion title="npm install Fails">
    If dependencies fail to install:

    ```bash
    # Clear npm cache
    npm cache clean --force
    
    # Delete node_modules and package-lock.json
    rm -rf node_modules package-lock.json
    
    # Reinstall
    npm install
    ```
  </Accordion>

  <Accordion title="Compilation Errors">
    If you encounter TypeScript compilation errors:

    ```bash
    # Update Angular CLI
    npm install -g @angular/cli@latest
    
    # Reinstall dependencies
    rm -rf node_modules package-lock.json
    npm install
    
    # Clear Angular cache
    rm -rf .angular
    ```
  </Accordion>

  <Accordion title="Blank White Screen">
    If the application loads but shows a blank screen:

    1. Open browser developer tools (F12)
    2. Check the Console tab for JavaScript errors
    3. Check the Network tab for failed API requests
    4. Verify backend API is running and accessible
    5. Check CORS configuration if using external API
  </Accordion>
</AccordionGroup>
